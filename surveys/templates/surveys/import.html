Markdown format

One # heading for the survey title, then survey metadata, then one ## heading per question with its properties as - key: value lines. 

# Post-event feedback
slug: post-event
status: draft

## How would you rate the event?
- type: rating
- max: 5
- labels: {"1": "Poor", "5": "Excellent"}

## Which sessions did you attend?
- type: multi_select
- choices: ["Talks", "Workshops", "Hallway"]
- required: false

## How likely are you to recommend us?
- type: nps

## Anything else?
- type: open_text
- required: false

## Would you come again?
- type: yes_no
  • type is required. One of: rating, multi_select, nps, open_text, yes_no.
  • required defaults to true; set required: false to make a question optional.
  • JSON values (lists, objects) are parsed as JSON. Bare numbers parse as integers.
  • slug defaults to a slugified title if you omit it. status defaults to draft.
